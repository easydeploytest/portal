Step 1 of 3
Start from the template.
Skip the boilerplate.

The template already has a Dockerfile, a CI workflow, and a /healthz endpoint. You write your app — not the scaffolding.


Click Use this template → Create a new repository inside the easydeploytest org. Or hit the green button in the top-right and the portal does it for you.

Open template ↗
your-repo/
├── src/           ← your code goes here
├── Dockerfile     ← already there
├── app.yaml       ← you edit this (next step)
└── RUNBOOK.md     ← auto-generated on deploy
RUNBOOK.md is generated on every deploy — all your URLs, secrets link, dashboard link, rollback instructions. Always current.
Step 2 of 3
Three lines.
That's the entire config.

Edit app.yaml. Set your app's name, team, and port.


That's the only platform file you write. From those three lines the platform provisions:

  • ✅  HTTPS endpoints for dev and prod
  • ✅  TLS certificate — auto-renewed
  • ✅  Secrets vault, isolated per app
  • ✅  Grafana dashboard with traces, logs, metrics
  • ✅  ArgoCD GitOps pipeline
  • ✅  Deploy notifications
app.yaml 
name: your-app   # unique, lowercase, ≥5 chars
team: easy-deploy
port: 3000

Step 3 of 3
Push to main.
It's live in ~2 minutes.

Commit your code, push to main. CI fires automatically and you'll get a notification when the app is healthy.


Prod? Publish a GitHub Release tagged v1.0.0. Platform re-tags the image and syncs prod.


Secrets? Add env vars in Infisical → your project. Injected within minutes, no redeploy.
