Step 1 of 3
Start from the template.
Skip the boilerplate.

The template repo already has a Dockerfile, a CI workflow, and a health endpoint. You don't need to configure any of that.


Click Use this template → Create a new repository — make sure the repo is inside the easydeploytest org so CI picks up the shared credentials automatically.


That's your only prerequisite. No platform tickets. No access requests.

Open template on GitHub ↗
your-repo/ 
├── src/           ← your code goes here
├── Dockerfile     ← already there
├── app.yaml       ← you edit this (next step)
└── RUNBOOK.md     ← auto-generated on deploy
The RUNBOOK.md is generated automatically on every deploy — all your URLs, secrets link, dashboard link, and rollback instructions. Always up to date.
Step 2 of 3
Three lines.
That's the entire config.

Edit app.yaml. Set your app's name, the team it belongs to, and the port your HTTP server listens on.


That's the only platform file you write. From those three lines the platform generates:

  • ✅  HTTPS endpoints for dev and prod
  • ✅  TLS certificate — auto-provisioned, auto-renewed
  • ✅  Secrets vault, isolated per app
  • ✅  Grafana dashboard with traces, logs, metrics
  • ✅  ArgoCD GitOps deployment pipeline
  • ✅  Deploy notifications
app.yaml 
name: your-app   # unique, lowercase
team: easy-deploy
port: 3000        # whatever your server uses

Step 3 of 3
Push to main.
It's live in ~2 minutes.

Commit your code, push to main. That's the whole workflow.


CI fires automatically, dispatches to the EasyDeploy platform, and you'll get a notification when your app is healthy.


Deploying to prod? Publish a GitHub Release with a tag like v1.0.0. The platform re-tags your image and ArgoCD syncs prod automatically.


Secrets? Go to Infisical → your project → add env vars. They're injected into your pods within minutes — no redeploy needed.
